## 4. CICLO DE VIDA DE UNA SOLICITUD
???????????????????????????????????????????????????????????????????????????????

1?? INICIO DEL WORKER SERVICE
   ?? Program.cs inicia Host
   ?? ConfigureServices() registra todas las dependencias
   ?? Host.RunAsync() inicia el worker
   ?? Worker.ExecuteAsync() comienza ejecuciÛn

2?? INYECCI”N DE DEPENDENCIAS EN WORKER
   ?? Container resuelve Worker constructor
   ?? Inyecta ILogger<Worker>
   ?? Inyecta ICsvReaderService
   ?? Inyecta IEtlService
   ?? Inyecta IHostApplicationLifetime

3?? RESOLUCI”N DE DEPENDENCIAS EN ETLSERVICE
   ?? Container crea nueva instancia de EtlService
   ?? Container crea CiudadRepository (crea DbContext tambiÈn)
   ?? Container crea PersonaRepository (reutiliza DbContext de scope)
   ?? Container crea GatoRepository (reutiliza DbContext)
   ?? Container crea FechaRepository (reutiliza DbContext)
   ?? Todas las instancias se usan juntas en el scope

4?? EJECUCI”N DEL ETL
   ?? Usa CsvReaderService para leer
   ?? Usa EtlService para transformar
   ?? EtlService usa los 4 repositorios
   ?? Los repositorios usan GenericRepository
   ?? GenericRepository usa DbContext compartido

5?? LIBERACI”N DE RECURSOS
   ?? Al finalizar el scope, se libera DbContext
   ?? Se cierran conexiones de base de datos
   ?? Se libera memoria
   ?? IDisposable se ejecuta autom·ticamente
